Working today

  • Published forms with document upload, submission ID and confirmation email.
  • API handoff to CaMS with the program code; cases file themselves into the right team's queue.
  • Request-more-information loop with the response logged in the case timeline.
  • Status changes (Received → Accepted / Rejected / Archived), archive, CSV export, teams & access controls.

To build — for this to work well

  • CaMS owns communication: one reference ID, staff notified on handoff, applicant emailed on every status change — rejection emails with contact details.
  • A front door: the tracking link shared in the confirmation email and a citizen side to check status.
  • Physical items handled honestly: digital copies by email, a “Ready for collection” state after Approved, no archiving until the original is picked up.
  • Case workflow: roles & permissions for managing / reviewing / approving, a redesigned case detail page, document upload in responses.
  • Looking ahead: status feedback to the forms app to catch duplicate applications.
